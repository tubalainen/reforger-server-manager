## What's new in 0.43.3

### Windows scripts now update themselves on start
Until now, clicking the Desktop shortcut pulled the newest manager **image** but never
refreshed the Windows helper scripts (`start.ps1`, `common.ps1`, …). So a fix to the scripts
themselves — like the v0.43.2 health-check fix (#135) — only reached you if you re-ran the
installer by hand.

`start.ps1` now refreshes the scripts from GitHub at the start of every launch:

- It downloads the current scripts, and swaps in only the ones that actually changed.
- If `start.ps1` itself changed, it relaunches the new copy so the fix applies on the same
  click.
- It's resilient: an offline launch just keeps the scripts already on disk, and a partial
  download never leaves a half-updated set.
- Skipped by `-NoUpdate` (which already skips the image pull too), or `-NoSelfUpdate` to skip
  only the script refresh.

Your `.env` is never touched. The compose file still updates via the installer.

**One-time upgrade note:** because the *old* `start.ps1` is what runs when you click the
shortcut, this self-update only kicks in once you're on a `start.ps1` that has it. Re-run the
Windows installer once to pick it up; from then on the scripts keep themselves current.

**Upgrade:** the manager updates itself on start, or pull
`ghcr.io/tubalainen/reforger-server-manager:latest`.
