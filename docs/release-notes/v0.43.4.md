## What's new in 0.43.4

### Deleting an instance can now wipe its stored data too
Deleting a server instance removed its container and its entry in the manager, but left the
instance's folder on disk — the baked mods, the saved game, logs and configs — behind. Worse,
once the instance was gone there was no in-app way to reach that folder again, so the leftover
data could only be cleared by hand on the host.

The delete dialog now shows what's on disk (per-category sizes) and offers a **"delete all
stored data from disk"** opt-in:

- Leave it unticked (the default) and nothing changes — the container comes off and the data
  stays, exactly as before.
- Tick it and the whole instance folder is removed as part of the delete, so nothing is left
  orphaned. Because that includes the **saved game**, the dialog warns clearly that the
  persistent world is gone for good.

A wipe that fails now surfaces as an error instead of being silently swallowed.

**Upgrade:** the manager updates itself on start, or pull
`ghcr.io/tubalainen/reforger-server-manager:latest`.
