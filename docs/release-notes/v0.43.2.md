## What's new in 0.43.2

### Fix: Windows start script wrongly reported the manager as "down" (#135)
The Windows start/update script's "is the manager answering?" check could stall for up to
a minute and a half and then warn that the manager *did not answer* — even when it was
already up and reachable in the browser.

The check used `Invoke-WebRequest` against `localhost`, which on Windows is routed through
the system proxy (WPAD auto-detect usually does **not** bypass localhost) and resolves to
IPv6 `::1` first, where Docker Desktop's published port may not be listening. Both made the
probe time out on every attempt.

The probe now connects directly to `127.0.0.1` with the proxy bypassed, retries with a
visible progress indicator, and reports success within a second or two. The web GUI address
shown to you is unchanged (`http://localhost:<port>`).

**Upgrade:** re-run the Desktop shortcut (it updates the manager on start), or pull
`ghcr.io/tubalainen/reforger-server-manager:latest`. To pick up the fixed start script
itself, re-run the installer or update your local `scripts/windows` copy.
